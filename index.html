// SETUP — DO THIS ONCE
1
ENABLE CF EMAIL ROUTING
dash.cloudflare.com → your domain → Email → Email Routing → Enable. CF auto-adds MX + SPF DNS records. Accept them.
2
VERIFY DESTINATION ADDRESS
Email Routing → Destination addresses → add your real inbox (e.g. you@gmail.com). Click the verification link CF sends you.
3
CREATE SCOPED API TOKEN
Profile → API Tokens → Create Token → Custom Token.
Permissions: Zone → Email Routing Rules → Edit + Zone → Zone → Read. Scope to your zone. Paste token in Config tab.
4
GET ZONE ID
Cloudflare dashboard → your domain → scroll the Overview right sidebar → copy Zone ID (32-char hex) → paste in Config tab.
5
ENABLE CATCH-ALL (RECOMMENDED)
In the Generate tab, flip the CATCH-ALL toggle ON. This creates one rule in CF that forwards everything to your inbox. You can then use any alias you invent on the spot — no rule creation needed.
// SEND FROM YOUR ALIAS (OPTIONAL — FOR REPLIES)
A
FREE: GMAIL "SEND MAIL AS"
Gmail → Settings → Accounts → Add another email address. Enter your alias. Use Gmail's SMTP (smtp.gmail.com:587) with an App Password. Caveat: headers say "via gmail.com" — some sites detect this.
B
PROPER: RESEND.COM + CF EMAIL WORKER (FREE TIER)
1. Sign up at resend.com (free: 3,000 emails/mo, 100/day)
2. Add & verify your domain in Resend (adds DKIM + SPF)
3. Use Resend API / SMTP to send from alias@yourdomain.com
4. Replies come back via CF Email Routing to your real inbox
Result: perfect SPF + DKIM pass, no "via" header, looks 100% legit.
C
ADVANCED: CLOUDFLARE EMAIL WORKERS
Use CF Email Workers to process inbound mail programmatically — filter spam, auto-reply, pipe to webhooks, etc. Free within CF Workers limits.
